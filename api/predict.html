Creates a model with defaulted values without need to perform an hyperparameter search at all times. Its initutive to have run the hyperparameter search beforehand to know the hyperparameter value to set. :param dense_layers: number of layers for the model architecture e.g for a model with 3 layers, values will be passed as [8,20,30] :param activation: activation function to be used e.g relu, tanh :param optimizer: optimizer to be used in compiling the model e.g relu, rmsprop, adam :param dropout_rate: used to make the model avoid overfitting, value should be less than 1 e.g 0.3 :param length: length of the trainset :param learning_rate: learning rate determines how fast or how slow the model will converge to an optimal loss value. Value should be less or equal 0.1 e.g 0.001 :param epochs: number of iterations the model should perform :param X_train: X trainset :param y_train: y trainset :param X_test: X testset :param y_test: y testset :param y_test_complete: dataframe containing the target variable with corresponding datapointid for the test set :param scalery: y scaler used to transform the y values to the original scale :param X_validate: X validation set :param y_validate: y validation set :param y_validate_complete: dataframe containing the target variable with corresponding datapointid for the validation set :param output_path: Where the outputs will be placed/

Returns

evaluation results containing the loss value from the testset prediction, annual_metric: predicted value for each datapooint_id is summed to calculate the annual energy consumed and the loss value from the testset prediction, output_df: merge of y_pred, y_test, datapoint_id, the final dataframe showing the model output using the testset val_metric:evaluation results containing the loss value from the validationset prediction, val_annual_metric:predicted value for each datapooint_id is summed to calculate the annual energy consumed and the loss value from the validationset prediction,, output_val_df: merge of y_pred, y_validate, datapoint_id, the final dataframe showing the model output using the validation set

predict.main(config_file=<typer.models.ArgumentInfo object>, preprocessed_data_file=<typer.models.ArgumentInfo object>, selected_features_file=<typer.models.ArgumentInfo object>, perform_param_search=<typer.models.OptionInfo object>, output_path=<typer.models.OptionInfo object>, random_seed=<typer.models.OptionInfo object>)¶

Select the feature which contribute most to the prediction for the total energy consumed. Default estimator_type used for feature selection is ‘LassoCV’

Parameters

  • config_file (str) – Location of the .yml config file (default name is input_config.yml).

  • preprocessed_data_file (str) – Location and name of a .json preprocessing file to be used.

  • selected_features_file (str) – Location and name of a .json feature selection file to be used.

  • perform_param_search (str) – ‘yes’ if hyperparameter tuning should be performed (increases runtime), ‘no’ if the default hyperparameters should be used.

  • output_path (str) – Where output data should be placed. Note that this value should be empty unless this file is called from a pipeline.

  • random_seed (int) – Random seed to be used when training.
